# UX/UI & AI 기획 문서 HTML 프롬프트 템플릿

아래 프롬프트를 그대로 복붙해서 사용하면 됩니다.

## 복붙용 프롬프트

```md
이 프로젝트는 html 생성기 프로젝트야.

다음 조건으로 결과물을 만들어줘.

1. 결과물 형태
- 단일 `html` 파일로 만들어줘
- `css`와 `js`는 절대 분리하지 말고 반드시 `html` 내부에 같이 작성해줘
- 바로 실행 가능한 형태로 만들어줘

2. 화면 사이즈
- 기본 작업 사이즈는 `1920px x 1080px`
- 프레젠테이션 슬라이드 또는 기획 문서 보드처럼 보이게 구성해줘
- 큰 화면에서 비율이 깨지지 않게 만들어줘

3. 디자인 방향
- 전체 톤앤매너는 내가 공유한 레퍼런스 이미지 느낌을 기본으로 해줘
- 화이트 배경, 볼드한 타이포, 깔끔한 카드형 레이아웃, 임팩트 있는 포인트 컬러, 프레젠테이션 템플릿 같은 인상을 반영해줘
- 문서 성격은 `UX/UI & AI 서비스 기획자 기획 문서`처럼 구성해줘
- 너무 개발 문서처럼 딱딱하지 않고, 전략 문서 + 제안서 + 서비스 소개서의 중간 톤으로 만들어줘

4. 폰트
- `Pretendard`를 기본 폰트로 사용해줘

5. 콘텐츠 방향
- 커버 섹션
- 서비스 소개 섹션
- 문제 정의 섹션
- UX 전략 섹션
- AI 기능 정의 섹션
- 사용자 플로우 또는 핵심 기능 섹션
- KPI / 기대효과 섹션
- 로드맵 섹션

6. 구현 방식
- 반응형도 고려하되 기본 기준은 반드시 `1920x1080`
- 더미 문구는 실제 서비스 기획 문서처럼 자연스럽게 작성해줘
- 시각적으로 보기 좋은 배치와 계층 구조를 잡아줘
- 필요하면 슬라이드형 탐색 또는 섹션 전환 인터랙션도 넣어줘

7. 산출물 품질
- 디자인 퀄리티를 높게 잡아줘
- 바로 실무에서 시안으로 보여줄 수 있는 수준으로 만들어줘
- 코드도 너무 난잡하지 않게 정리해줘
```

## 빠르게 수정해서 쓰는 버전

```md
이 프로젝트는 html 생성기 프로젝트야.
톤앤매너는 내가 공유한 레퍼런스 이미지를 기반으로 하고,
기본 사이즈는 1920px x 1080px야.

`UX/UI & AI 서비스 기획 문서` 스타일로 만들어줘.
폰트는 Pretendard를 사용하고,
css와 js는 따로 분리하지 말고 html 내부에 같이 작성해줘.

화이트 배경, 큰 타이포, 카드형 레이아웃, 발표용 슬라이드 같은 느낌으로 구성하고,
커버 / 서비스 소개 / 문제 정의 / UX 전략 / AI 기능 / KPI / 로드맵 섹션을 포함해줘.

실무 시안처럼 퀄리티 있게 만들어줘.
```
