AI Service Planning Framework

AI 서비스 기획 내용을 4단계 실행 프레임워크로 재구성

주신 체크리스트를 기준으로 흐름이 자연스럽게 이어지도록 4단계 구조로 정리했습니다. `Evaluation`과 `Operations`는 실제 운영 고도화 관점에서 함께 보는 경우가 많아 마지막 단계로 통합했습니다.
1. 비즈니스 및 비용 설계

Economics

핵심 체크리스트
Core Checklist
  • AI 적합성 판단: 규칙 기반(If-else)으로 해결 가능한지, 반드시 LLM이 필요한지 검토
  • Unit Economics 산출: 사용자 1인당 평균 토큰 소모량 vs 예상 매출(LTV) 계산
  • 수익 모델 기획: API 비용 변동에 대응할 구독 플랜 또는 종량제 모델 설계
  • 데이터 확보 계획: RAG에 활용할 내부 데이터 자산 리스트업 및 저작권 검토
2. 기술 검토 및 모델 선정

Feasibility

핵심 체크리스트
Core Checklist
  • 모델 벤치마킹: GPT, Claude, Gemini, Llama 등 모델별 성능/비용/속도 비교 시트 작성
  • 토큰 최적화 전략: Context Window 관리 및 비용 절감을 위한 프롬프트 압축 방안 수립
  • 데이터 파이프라인 설계: PDF, 텍스트 등 비정형 데이터를 AI가 읽기 좋게 쪼개는 Chunking 규칙 정의
  • 외부 API vs 자체 모델(sLLM) 판단: 보안 및 비용 관점의 인프라 구조 결정
3. 서비스 및 UX 기획

Product Design

핵심 체크리스트
Core Checklist
  • System Prompt 설계: AI의 페르소나, 말투, 답변 형식, 제약 사항 정의
  • AI 특화 UX 설계: 스트리밍 답변 UI, 답변 재생성(Regenerate), 중단(Stop) 기능 기획
  • 에러 핸들링: 할루시네이션 방지 문구 및 AI 답변 오류 시 신고/피드백 버튼 배치
  • 가드레일 설정: 금지어 필터링 및 민감 정보 마스킹 로직 정의
4. 성능 평가 및 운영 고도화

Evaluation + Operations

핵심 체크리스트
Evaluation
  • 골든 셋 구축: 정답으로 간주할 질문-답변 쌍 데이터셋 최소 50~100개 확보
  • 정성/정량 지표 수립: 답변 정확도, 유저 만족도, 응답 속도 등 핵심 KPI 설정
  • 피드백 루프 기획: 유저의 좋아요/싫어요 데이터를 모델 개선에 반영하는 프로세스 설계
  • A/B 테스트: 프롬프트 버전별(v1 vs v2) 전환율 및 리텐션 비교 분석
Operations
  • Rate Limit 설정: 특정 유저의 과도한 API 호출을 막기 위한 사용량 제한 기획
  • 프롬프트 버전 관리: 성능 변화에 따른 프롬프트 히스토리 관리 체계 수립
  • 보안 및 규제 대응: 데이터 서버 위치(국내외) 및 AI 윤리 가이드라인 준수 여부 체크
