분리된 협업툴을 넘어서
대화 자체가 자산이 되는 브레인 시스템

1.1 전략 정의

협업 과정 전체를 하나의 데이터 흐름으로 기록하는 구조 설계

채팅, 문서, AI, 실행이 분리된 업무 환경을 통합해 대화 → 아이디어 → 문서 → 스펙 → 실행이 연결되는 AI 개입형 협업 구조를 목표로 설정했습니다.

1.2 문제 정의

브레인스토밍은 진행되지만, 사고 흐름은 축적되지 않았습니다

아이디어는 메신저에 흩어지고 정리는 다시 문서로 작성되며 AI는 개별 도구로 호출되는 구조여서, 의사결정 인과관계 추적과 지식 축적이 어려운 문제가 존재했습니다.

01

기존 채팅, 문서, AI툴의 분리 구조에서 벗어나 대화 자체가 기록되고 구조화되며 결과로 이어지도록 시스템을 설계했습니다.

02

대화 → 아이디어 → 문서 → 스펙 → 실행 흐름 구성

브레인스토밍 중 생성되는 대화, 결정, 아이디어가 자동 정리되고 문서와 스펙으로 전환될 수 있는 데이터 흐름을 만들었습니다.

03

로그 기반 추적 구조 설계

어떤 논의가 어떤 결과로 이어졌는지 추적할 수 있도록 협업 로그와 의사결정 인과관계 분석 구조를 함께 설계했습니다.

04

AI 역할 재정의

AI가 요약, 구조화, 문서 생성, 스펙 생성, 지식 정리까지 수행하는 협업 개입형 역할을 하도록 제품 방향을 정의했습니다.

Project 04 / Dongwha AI Orchestration Platform (DAOP)

데이터, 정책, 액션, AI를 연결해
전사 자동화를 조립하는 플랫폼

4.1 전략 정의

현업 사용자가 직접 자동화를 조립할 수 있는 AI Orchestration Platform 기획

분산된 시스템과 데이터, 문서, 프로세스를 연결하고 업무 자동화와 AI 활용을 전사 차원으로 확장할 수 있는 플랫폼을 목표로 설정했습니다.

4.2 문제 정의

자동화 로직이 시스템과 코드에 갇혀 현업이 직접 활용하기 어려웠습니다

데이터, 정책, 액션, 문서, API가 분리되어 있고 자동화가 개발 조직 중심으로 수행되어 확장성과 사용자 참여가 낮은 한계가 존재했습니다.
